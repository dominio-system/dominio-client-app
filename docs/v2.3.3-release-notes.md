### Feature nuevo

**Flechas back/forward en el topbar** (`dashboard.html` topnav-row1 + `app.js:navBack/navForward`)

- Stack de historial tipo browser. Cada `go(view)` empuja; las flechas mueven el puntero.
- Atajos: `⌘[` / `⌘]` (mac) y `Alt+←` / `Alt+→` (cross-platform).
- Cap de 50 entradas en el stack.
- Flechas deshabilitadas con `:disabled` cuando no hay forward/back disponible.
- Se ignoran shortcuts si el foco está en input/textarea/contentEditable.

### Panel · Próximas Citas (reemplazo de ARIA Sugerencias)

**Ubicación:** columna derecha del grid Resumen (debajo del KPI strip).

**Altura locked:** 480px. Chart y panel **siempre paralelos** sin importar cantidad de citas.

**Responsive (Electron min 900, default 1280):**

| Ancho ventana | Grid chart : panel | Mini-stats panel |
|---------------|--------------------|------------------|
| ≥ 1800px      | `1fr : 460px`      | 2 columnas       |
| 1400 – 1799px | `1fr : 420px`      | 2 columnas       |
| 1200 – 1399px | `1fr : 400px`      | 2 columnas       |
| 1000 – 1199px | `1fr : 360px`      | 2 columnas       |
| 900 – 999px   | `1fr : 340px`      | **1 columna apilada** |

**Estructura del panel (4 zonas):**

1. **Header** (estático) — Título "ARIA agendó" + chip live verde pulsante "N citas"
2. **Mini KPIs** (estático) — Captación promedio (min) + Cierre por ARIA (%)
3. **Lista** (scroll interno con fades top/bottom) — Cada cita: avatar inicial · nombre · tag HOY/MAÑANA (verde/gris-azulado) · hora · servicio · botones ⓘ y 💬
4. **Footer** (estático) — Live · sync HH:MM + link Ver agenda →

**Modales:**

- **Info (ⓘ):** Timeline vertical con 3 pasos: Primer mensaje · Cita confirmada por ARIA · Cita programada. Card destacada con tiempo de captación (diff entre primer mensaje WhatsApp del lead y `appointments.created_at`).
- **Chat (💬):** Hilo completo WhatsApp · burbujas in/out · timestamps · card final "✓ Cita agendada". Botón "Abrir en ARIA →" navega a la sección ARIA.

**Realtime:** ya wired vía `realtime.js:handleAppointmentChange` que debouncea `appt-brief` → `loadBriefData` → `loadProximasCitas`. INSERT/UPDATE en `appointments` actualiza el panel sin refresh manual.
